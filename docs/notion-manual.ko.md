# Second Claude Code 사용 매뉴얼

> Fronmpt Academy 수강생을 위한 실전 가이드

---

## 1. Second Claude Code가 뭔가요?

**한 줄 요약**: Claude Code 위에 올리는 플러그인이에요. 리서치, 글쓰기, 분석, 리뷰, 개선을 프롬프트 한 줄로 끝까지 돌려줘요.

지금까지는 이렇게 하셨을 거예요:

| 기존 방식 | Second Claude Code |
|---|---|
| ChatGPT에서 리서치 → 복사 → 다른 창에서 글쓰기 → 복사 → 또 다른 창에서 피드백 요청 | "AI 에이전트 알아보고 보고서 써줘" 한 줄이면 리서치부터 리뷰까지 끝 |
| 초안 쓰고 → 동료한테 피드백 요청 → 하루 기다리기 | 전문 리뷰어 5명이 30초 만에 병렬 검토 |
| SWOT 분석 양식 찾고 → 빈칸 하나씩 채우기 | "SWOT으로 분석해줘" 한 마디면 15개 프레임워크 중 자동 선택 |
| 글 퀄리티가 아쉬워서 프롬프트 3~4번 반복 | 목표 점수 설정하면 알아서 반복 개선 |

핵심은 **컨텍스트 스위칭이 사라진다**는 거예요. 리서치 결과가 글쓰기로 자동으로 넘어가고, 초안이 리뷰로 자동으로 넘어가고, 피드백이 자동으로 반영돼요. 사이사이를 잇는 복사-붙여넣기가 없어요.

## 2. 설치 (3분)

### 전제 조건

Claude Code가 먼저 설치되어 있어야 해요.
아직 없다면: [Claude Code 공식 설치 가이드](https://docs.anthropic.com/en/docs/claude-code/overview)

### 설치 명령어

터미널에서 이 한 줄을 실행하세요:

```bash
claude plugin add github:unclejobs-ai/second-claude-code
```

### 확인 방법

Claude Code를 새 세션으로 열면 이 메시지가 보여야 해요:

```
# Second Claude Code
9 commands for all knowledge work:
```

안 보이면 `claude plugin list`로 플러그인이 제대로 설치됐는지 확인하세요.

## 3. 이런 상황에서 쓰세요

### 용례 1. 시장 조사 보고서 작성

**상황**: 새 사업 아이템을 검토 중인데, 시장 현황을 정리한 보고서가 필요해요.

**프롬프트**:

```
국내 AI 교육 시장 현황을 조사하고 보고서를 작성해줘.
시장 규모, 주요 플레이어, 성장 전망을 포함해줘.
```

**결과**: 20개 이상 소스를 크롤링한 리서치 브리프 + 3,000자 이상 구조화된 보고서 + 리뷰어 5명의 검증 리포트가 나와요.

**기존 대비**: ChatGPT로 하면 리서치 따로, 글쓰기 따로, 검증은 직접. 최소 3~4번의 프롬프트와 복사-붙여넣기가 필요했어요. 여기선 프롬프트 1번이에요.

---

### 용례 2. 경쟁사 SWOT 분석

**상황**: 투자 미팅 전에 경쟁사 분석 자료가 급하게 필요해요.

**프롬프트**:

```
클래스101, 패스트캠퍼스, 인프런을 SWOT으로 비교 분석해줘.
우리 서비스와의 차별점도 정리해줘.
```

**결과**: 15개 내장 프레임워크(SWOT, Porter, RICE 등) 중 적합한 것을 자동 선택해서 구조화된 전략 분석이 나와요. 프레임워크를 직접 지정할 수도 있어요.

**기존 대비**: SWOT 템플릿 찾고, 각 항목 따로 리서치하고, 표 정리하고. 30분~1시간 걸리던 작업이 프롬프트 1번으로 줄어요.

---

### 용례 3. 뉴스레터/블로그 글 작성

**상황**: 이번 주 뉴스레터 주제가 정해졌는데 초안을 빠르게 뽑고 싶어요.

**프롬프트**:

```
"비개발자가 AI 에이전트를 활용하는 3가지 방법"이라는 주제로
뉴스레터를 작성해줘. 타겟은 AI에 관심 있는 직장인이야.
```

**결과**: 자동으로 주제를 리서치하고, 뉴스레터 포맷(Hook, 3~4개 섹션, 실행 가능한 마무리)에 맞춰 2,000자 이상 초안이 나와요. 리뷰어가 논리, 톤, 팩트를 검증한 뒤 넘겨줘요.

**기존 대비**: 리서치 30분 + 초안 작성 1시간 + 퇴고 30분. 2시간짜리 작업이 프롬프트 1번으로 됩니다.

---

### 용례 4. 기존 초안 품질 검증

**상황**: 제안서 초안을 다 썼는데, 내보내기 전에 빈틈이 없는지 확인하고 싶어요.

**프롬프트**:

```
이 제안서 초안을 리뷰해줘. 논리적 허점, 누락된 근거, 톤 불일치를 중심으로.

[초안 내용을 붙여넣거나 파일 경로 지정]
```

**결과**: 전문 리뷰어 5명이 병렬로 초안을 검토해요:
- 네이티오(Xatu): 논리 흐름과 완결성
- 앱솔(Absol): 가장 약한 지점을 찾아서 공격
- 폴리곤(Porygon): 숫자, 주장, 출처 팩트체크
- 푸린(Jigglypuff): 어조 일관성, 독자 적합성
- 안농(Unown): 가독성, 문서 구조

2/3 이상이 통과해야 "승인"이 나와요. 치명적 문제가 하나라도 있으면 차단돼요.

**기존 대비**: 동료에게 리뷰 부탁하면 하루 이상 걸리고, 관점도 1명분이에요. 여기선 5개 관점을 30초 안에 받아요.

---

### 용례 5. 초안 다듬기 (목표 품질까지)

**상황**: 리뷰 결과가 나왔는데 "아직 부족하다"고 해요. 통과할 때까지 개선하고 싶어요.

**프롬프트**:

```
이 초안을 리뷰어가 통과할 때까지 다듬어줘. 최대 3회 반복.
```

**결과**: 리뷰 피드백 중 우선순위 상위 3개를 반영 → 다시 리뷰 → 개선 → 다시 리뷰... 목표 점수에 도달하거나 최대 횟수에 도달하면 멈춰요. 품질이 안 오르면 자동으로 중단해서 토큰을 낭비하지 않아요.

**기존 대비**: "이거 고쳐줘" → "이것도 고쳐줘" → "아까 말한 것도 반영해줘" 수동 반복이 사라져요.

---

### 용례 6. 매주 반복하는 콘텐츠 워크플로우 자동화

**상황**: 매주 같은 패턴으로 뉴스레터를 쓰고 있어요. 리서치 → 글쓰기 → 리뷰를 매번 따로 시키기 귀찮아요.

**프롬프트**:

```
"주간 뉴스레터"라는 워크플로우를 만들어줘.
1단계: 주제를 리서치
2단계: 뉴스레터 포맷으로 작성
3단계: 콘텐츠 프리셋으로 리뷰
4단계: 피드백 반영해서 다듬기
```

만들어두면 이후에는 이렇게 실행해요:

```
주간 뉴스레터 워크플로우 실행해줘. 이번 주 주제는 "프롬프트 엔지니어링 트렌드"야.
```

**결과**: 저장된 워크플로우가 4단계를 자동으로 순서대로 실행해요. 주제만 바꿔서 매주 재사용할 수 있어요.

**기존 대비**: 매주 같은 프롬프트 4개를 순서대로 치는 반복 노동이 사라져요.

---

### 용례 7. 새 기능이 필요할 때 (스킬 탐색)

**상황**: "이메일 마케팅 카피를 써주는 기능 없나?" 싶을 때.

**프롬프트**:

```
이메일 마케팅 카피를 도와주는 스킬이 있는지 찾아봐.
```

**결과**: 먼저 기존 9개 스킬로 가능한지 확인하고, 부족하면 외부 마켓플레이스에서 후보를 검색해요. 관련성, 인기도, 안전성을 점수로 매겨서 추천해줘요. 설치는 승인 후에만 진행돼요.

**기존 대비**: 직접 플러그인 마켓을 뒤지고, 리뷰 읽고, 호환성 확인하는 시간이 절약돼요.

## 4. 9개 스킬 한눈에 보기

| 이걸 하고 싶으면 | 스킬 | 설명 |
|---|---|---|
| 주제를 깊이 파고 싶을 때 | **조사** (`research`) | 20개 이상 소스 크롤링, 패턴 분석, 브리프 생성 |
| 전략 프레임워크로 분석할 때 | **분석** (`analyze`) | SWOT, Porter, RICE 등 15개 프레임워크 지원 |
| 아티클/보고서/뉴스레터를 쓸 때 | **글쓰기** (`write`) | 리서치 기반 초안 + 자동 리뷰 포함 |
| 초안을 여러 관점에서 검증할 때 | **검증** (`review`) | 전문 리뷰어 5명 병렬 검토, 합의 투표 |
| 피드백 반영해서 목표까지 다듬을 때 | **다듬기** (`refine`) | 리뷰-수정 반복, 목표 점수 도달 시 자동 종료 |
| URL, 메모, 자료를 저장할 때 | **수집** (`collect`) | PARA 분류 기반 지식 캡처 |
| 여러 스킬을 자동화할 때 | **워크플로우** (`workflow`) | 스킬 체이닝, 저장/재사용 가능 |
| 새 스킬을 찾고 싶을 때 | **탐색** (`discover`) | 마켓플레이스 검색, 점수 평가, 승인 후 설치 |
| 리서치부터 퍼블리시까지 전체를 돌릴 때 | **전체 사이클** (`pdca`) | Plan-Do-Check-Act 자동 순환 |

자연어로 말하면 자동 라우팅이 맞는 스킬을 골라줘요. 정확도 약 95%. 직접 지정하고 싶으면 슬래시 명령어(`/second-claude-code:write` 등)를 쓸 수도 있어요.

## 5. 자주 묻는 질문

### Claude Code가 뭔가요?

Anthropic이 만든 터미널 기반 AI 코딩/작업 도구예요. ChatGPT 같은 웹 채팅이 아니라 터미널에서 동작하고, 파일을 직접 읽고 쓸 수 있어요. Second Claude Code는 이 Claude Code 위에 올리는 플러그인이에요.

### 영어로 써야 하나요?

아니요. 한국어와 영어 모두 지원해요. 한국어 트리거 패턴이 약 50개, 영어가 약 77개 내장되어 있어요. 섞어 써도 자동으로 인식해요.

### 비용이 따로 드나요?

Second Claude Code 플러그인 자체는 무료(MIT 라이선스)예요. 다만 Claude Code 사용에 따른 API 비용은 기존과 동일하게 발생해요. 서브에이전트가 여러 개 돌아가니까 일반 프롬프트보다 토큰 사용량이 많을 수 있어요. 비용이 부담되면 `review_preset`을 `quick`(리뷰어 2명)으로 설정하거나, `research_depth`를 `shallow`로 낮추면 절약돼요.

### 다른 플러그인이랑 충돌하나요?

일반적으로 충돌하지 않아요. 다만 활성 플러그인이 많으면 컨텍스트 윈도우가 빡빡해질 수 있어요. 안 쓰는 플러그인은 비활성화하면 해결돼요.

### 한 번에 긴 글도 써지나요?

네. 3,000자 이상 장문 콘텐츠를 기본으로 지원해요. 보고서는 4,000자 이상, 아티클은 3,000자 이상이 기본 목표 분량이에요. `write` 스킬 내부에 opus 모델 기반 전문 라이터(루브도)가 장문 작성을 담당해요.

### 리뷰어가 왜 포켓몬 이름이에요?

각 포켓몬의 특성이 리뷰어 역할과 맞아떨어지기 때문이에요. 네이티오(Xatu)는 과거와 미래를 동시에 보는 포켓몬이라 구조적 결함을 잡고, 앱솔(Absol)은 재앙을 감지하는 포켓몬이라 취약점을 찾고, 폴리곤(Porygon)은 디지털 네이티브라 데이터 기반 팩트체크를 해요. 기억하기 쉽고, 기억하니까 누가 뭘 하는지 실제로 머릿속에 남아요.
