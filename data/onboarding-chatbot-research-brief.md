## 2. 기술 아키텍처 옵션

### 2.1 .md 파일 vs 벡터 DB — 규모별 판단 기준

| 조직 규모 | 문서 수 | 권장 방식 | 도구 | 이유 |
|---------|--------|---------|------|------|
| 스타트업 (1-10명) | ~100 | .md + 장문맥 주입 | Claude 1M 윈도우 | 인프라 불필요, 즉시 시작 |
| 소규모 (10-50명) | 100-500 | .md + BM25 검색 | CLAUDE.md / dotMD | 키워드 검색 충분 |
| 중견 (50-500명) | 500-5K | 하이브리드 (md+벡터) | LangChain + sqlite-vec | 의미론적 검색 필요 시작점 |
| 대규모 (500+명) | 5K+ | 전용 벡터 DB | Pinecone/Qdrant + Neo4j | 동시성, SLA 필수 |

**마이그레이션 신호**: 검색 정확도 < 70%, 5000+ 파일, 동시 업데이트 > 10명

### 2.2 토큰 비용 비교
| 방식 | 1,000회 일일 쿼리 토큰 | 월 비용 (Claude Sonnet) |
|------|----------------------|----------------------|
| 장문맥 전체 주입 (100K 토큰/회) | 1억 토큰 | ~$300,000 |
| 벡터 검색 최적화 (1K 토큰/회) | 100만 토큰 | ~$3,000 |
| **절감률** | **100배** | **99%** |

### 2.3 LLM API 비교
| 측면 | Claude API | OpenAI API |
|------|-----------|-----------|
| 입력 토큰 비용 (Sonnet/4o) | $3/M | $2.50/M |
| 출력 토큰 비용 | $15/M | $10/M |
| 컨텍스트 윈도우 | 200K (전 모델 동일) | 모델별 상이 |
| 강점 | 추론, 안정성, 장문 | 다재다능, 멀티모달 |
| Batch API 절감 | 50% | 50% |

**모델 선택 가이드 (온보딩용)**:
- **Haiku 4.5** ($0.80/$4/M): 단순 FAQ 응답, 높은 처리량 → Sonnet 90% 성능 @ 1/3 비용
- **Sonnet 4.5** ($3/$15/M): 일반 목적 추천, 품질/비용 최적 균형
- **오픈소스 (Ollama + Llama 3.1)**: 민감 데이터, 토큰 비용 $0, 인프라 비용만

## 4. 개인화 대화 설계 패턴

### 4.1 프로필링 프레임워크

신입사원을 3개 축으로 프로필링:

```
[역량 축]                    [성향 축]                    [맥락 축]
- 직무 경력 수준             - DiSC 성격 유형             - 입사 직무/부서
- 기술 스택 숙련도           - VAK 학습 스타일            - 근무 형태 (원격/출근)
- 자체 역량 평가             - 커뮤니케이션 선호          - 입사 시기 (수습/정규)
```

### 4.2 적응형 대화 분기

```
[초기 인터뷰 (5-10분)]
  "어떤 직무로 입사하셨나요?"
  "개발 경력은 어느 정도인가요?"
  "학습할 때 선호하는 방식은?"
       │
       ▼
[프로필 생성 → 경로 결정]
  ├── 주니어 + 부트캠프 출신
  │     → 기초 개념 포함, 용어 풀이, 단계별 가이드
  │     → 톤: 친근하고 격려적
  │
  ├── 시니어 + 경력자
  │     → 아키텍처/의사결정 배경 중심
  │     → 톤: 동료 수준, 핵심만
  │
  └── 직무 전환자
        → 도메인 맥락 브릿징, 이전 경험 연결
        → 톤: 공감적, 비교 설명
```

### 4.3 심리적 안전 메시징 원칙

| 원칙 | 구현 예시 |
|------|---------|
| **정상화** | "이 단계에서 헷갈리는 건 아주 자연스러운 거예요" |
| **성장 프레임** | "틀리는 것도 배우는 과정이에요" |
| **능동적 체크인** | "여기까지 이해되셨나요? 막히는 부분 있으세요?" |
| **실행 연결** | "지금 바로 해보실까요?" |
| **감정 인식** | 부정 감정 감지 → 톤 전환 + 멘토 연결 제안 |

### 4.4 게임화 마일스톤

| 단계 | 기간 | 미션 예시 | 보상 |
|------|------|---------|------|
| Level 1 | 1주차 | 시스템 접근, 팀 소개, 첫 PR | "First Commit" 배지 |
| Level 2 | 2-4주 | 직무별 스킬 태스크 완료 | "Quick Learner" 배지 |
| Level 3 | 1-3개월 | 전사 이해 미션, 크로스팀 협업 | "Culture Champion" 배지 |

## 6. 구현 로드맵

### Phase 1: 프로토타입 (1-2주)

```
[지식베이스 구축]         [챗봇 파이프라인]         [인터페이스]
.md 파일 20-30개 작성  →  LangChain + Chroma    →  Streamlit UI
YAML frontmatter 추가     한국어 임베딩 (Upstage)    또는 Slack 봇
                          Claude Haiku/Sonnet
```

**기술 스택**: Python + LangChain + Chroma + Claude API + Streamlit
**비용**: ~$100/월 (API 호출 기준)
**목표**: 10명 내부 테스트, 기본 Q&A 동작 확인

### Phase 2: 개인화 + 프로덕션 (2-4주)

```
[프로필 시스템]           [적응형 대화]             [통합]
초기 인터뷰 흐름 구현   →  레벨별 톤/깊이 조정    →  Slack/Teams 연동
프로필 .md 자동 생성      진행 추적 + 퀴즈           HR 시스템 연동
```

**추가 스택**: FastAPI + React + Weaviate (하이브리드 검색)
**비용**: ~$500/월
**목표**: 50명 파일럿, 개인화 대화 + 진행 추적

### Phase 3: 스케일업 (1-2개월)

```
[확장]                    [분석]                    [최적화]
벡터 DB 마이그레이션    →  대시보드 (만족도/진행률) →  A/B 테스트
멀티테넌트 (부서별)       감정 분석 + 에스컬레이션    게임화 마일스톤
```

**비용**: ~$1,000-3,000/월 (규모에 따라)
**목표**: 전사 배포, KPI 측정

| 지식베이스 노후화 | 자동 알림 + 주간 콘텐츠 리뷰 |

### 8.2 핵심 원칙
- 챗봇은 **정보 제공 보조 도구**로 역할 정의 (인간 멘토 대체 X)
- **"모르겠습니다"를 말할 수 있는 챗봇**이 할루시네이션보다 나음
- 고가치 계약($500K+) 또는 임원급 온보딩은 반드시 인간 상호작용 병행
