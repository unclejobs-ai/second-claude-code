# Journey Map Framework

## When to Use
- When analyzing stage-by-stage user experience and friction
- When investigating conversion, activation, or retention structurally

## When NOT to Use
- When user segments are not yet defined
- When the goal is to inspect only internal process flow

## Required Sections
1. Journey stages
2. User goals per stage
3. Pain points
4. Emotions / confidence
5. Opportunity areas

## Evidence Expectations
- Include behavioral evidence or observations for each stage
- Describe pain points with concrete situations

## Recommended Actions
- Recommend priorities for the 3 biggest friction points
