# Product Requirements Framework

## When to Use
- When structuring feature or product requirements
- When clarifying the problem, user, and success criteria before implementation

## When NOT to Use
- When the implementation plan is already fixed
- When only strategic analysis is needed

## Required Sections
1. Problem statement
2. Target user
3. Goals / non-goals
4. Requirements
5. Success metrics
6. Risks / open questions

## Evidence Expectations
- Tie requirements directly to the user problem
- Make goals and non-goals explicit

## Recommended Actions
- Recommend the 3 key decisions that must be locked before implementation
