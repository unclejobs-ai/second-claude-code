# Persona Framework

## When to Use
- When defining the core user types concretely
- When linking messaging, prioritization, or UX decisions to user context

## When NOT to Use
- When the work is only assumptions with no real user observation
- When the persona is just demographics with no behavioral context

## Required Sections
1. Persona summary
2. Goals and jobs
3. Behaviors and context
4. Frictions and needs
5. Buying / adoption triggers

## Evidence Expectations
- Include observation, interviews, or data-backed signals
- Prioritize behavior over demographics

## Recommended Actions
- Recommend 3 differentiation points by persona
